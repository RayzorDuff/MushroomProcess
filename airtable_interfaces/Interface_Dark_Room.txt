----------------------------------------

----------------------------------------
SECTIONS
----------------------------------------
1. Modify Status
   - Description: Actions for items remaining in the Dark Room
   - Buttons: Shake and Break, Fully Colonized

2. Move Inventory
   - Description: Actions for items moving from the Dark Room to the Fridge
   - Buttons: Cold Shock, Move to Fridge

3. Transition Inventory
   - Description: Actions for items moving out of the Dark Room
   - Buttons: Contaminated, Start Fruiting

4. Organism Introduction
   - Description: Actions for introducing organisms to the block substrate
   - Buttons: Apply Diatomaceous Earth, Add Nematodes

----------------------------------------

----------------------------------------
(Each button uses Airtable “Update record” actions targeting the current Record Detail.)

   - Label After: "Casing Applied"
   - Show check icon: true
   - Color: Default (gray to pink)
   - Record Source: Record Detail

----------------------------------------
NOTES ON FUNCTIONAL GROUPINGS
----------------------------------------
- Buttons are grouped logically to reflect workflow stages:
  • Modify Status: in-process changes within dark room
  • Move Inventory: physical relocation events
  • Transition Inventory: stage progression or contamination
  • Organism Introduction: biological control measures

- Each button updates “action” (and sometimes “status”) field in the `lots` table.
- Validation field appears informational and is not modified by buttons.
