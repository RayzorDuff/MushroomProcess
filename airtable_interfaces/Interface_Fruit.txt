----------------------------------------
SECTIONS
----------------------------------------
1. Transition Block
   - Description: Transition a block into or out of the fruiting chamber
   - Buttons: Compost Block, Start Fruiting

2. Organism Introduction
   - Description: Actions for introducing organisms to the block substrate
   - Buttons: Apply Diatomaceous Earth, Apply Casing, Apply Nematodes, Apply Beneficial Trichoderma

----------------------------------------
BUTTONS AND ACTION LOGIC
----------------------------------------
(Each button uses Airtable “Update record” actions targeting the current Record Detail.)

1. Compost Block
   - Set action = "Composted"
   - Set status = "Retired"
   - Label After: "Composted"
   - Show check icon: true
   - Color: Default (gray → yellow)
   - Record Source: Record Detail

2. Start Fruiting
   - Set status = "Fruiting"
   - Set action = "StartFruiting"
   - Label After: "Fruiting Started"
   - Show check icon: true
   - Color: Default (gray → purple)
   - Record Source: Record Detail

3. Apply Diatomaceous Earth
   - Set action = "ApplyDiatomaceousEarth"
   - Label After: "Diatomaceous Earth Applied"
   - Show check icon: true
   - Color: Default (gray → green)
   - Record Source: Record Detail

4. Apply Casing
   - Set action = "ApplyCasing"
   - Label After: "Casing Applied"
   - Show check icon: true
   - Color: Default (gray → pink)
   - Record Source: Record Detail

5. Apply Nematodes
   - Set action = "ApplyNematodes"
   - Label After: "Nematodes Applied"
   - Show check icon: true
   - Color: Default (gray → purple)
   - Record Source: Record Detail

6. Apply Beneficial Trichoderma
   - Set action = "ApplyBeneficialTrichoderma"

   - Record Source: Record Detail

----------------------------------------
NOTES ON FUNCTIONAL GROUPINGS
----------------------------------------
- **Transition Block section** handles major workflow transitions (into fruiting or retiring blocks).
- **Organism Introduction section** handles biological interventions within active fruiting operations.
- All buttons update “action” (and sometimes “status”) fields in the `lots` table.
- The “item_category = fruiting_block” filter ensures only fruiting-stage inventory appears.
- No advanced conditional visibility rules are applied (Visibility: Always enabled).
