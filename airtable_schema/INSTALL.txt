Step-by-Step Database Installation Guide
1. Create a New Airtable Base

Log into Airtable
Create a new base with any name (e.g. “Mushroom Process”).
Note the Base ID (e.g. appXXXXXXXXXXXXXX).

2. Set Up API Access

Visit https://airtable.com/account
Create a Personal Access Token with schema and data read/write permissions for the new base.
Note your token (e.g. patXXXXXXXXXXXXXX).

3. Configure Environment

Open a terminal and set the following environment variables:

export AIRTABLE_BASE=appXXXXXXXXXXXXXX
export AIRTABLE_TOKEN=patXXXXXXXXXXXXXX

On Windows (PowerShell):

$env:AIRTABLE_BASE = "appXXXXXXXXXXXXXX"
$env:AIRTABLE_TOKEN = "patXXXXXXXXXXXXXX"

4. Run the Setup Script

Ensure Node.js is installed (node -v to check).
From the airtable_schema/ directory, run:

node create_airtable_from_schema.js

This will:

Create each table listed in _schema.json
Add fields with the correct types, options, and relationships

Optional: Import Record Data

If you want to load some sample records (optional for testing):
Use airtable-export or a separate script to upload .json or .ndjson records into each table using the same table names.
This does not happen automatically in the schema script.

Notes:

You can rerun the script safely to create missing tables or fields.
It won’t delete or overwrite existing tables or records.
If you modify _schema.json, re-run the script to apply updates.
